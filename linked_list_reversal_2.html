What is a Node?

Each element is a node with two things: a value (the data) and a pointer to the next node. The last one points to null — the end.

👉
One Direction Only

Singly linked lists travel forward only. You can't jump back. To traverse backwards, you'd restart — which is why reversal matters!

🏠
vs. an Array

Arrays sit in one memory block. Linked list nodes scatter anywhere — each pointing to the next. Great for fast insertions/deletions, no random access.

🔄
The Goal: Reversal

Flip all the arrows. 1→2→3→4→5 becomes 5→4→3→2→1. The trick? We do it with just 3 variables in one pass.

02 — The Algorithm
How Does it Work?

We use exactly three pointer variables: prev, curr, and next. Here's each step in plain English.

0

Initialize the three pointers

prev = null — nothing behind us yet. curr = head — start at the first node. next = null — will be filled each loop.

prev = None | curr = head | next = None
1

💾 Save next before breaking the link

Before cutting the forward link, save it. If we don't, we'd lose the rest of the list — permanently. Always save first!

next = curr.next // save the next node
2

↩️ Flip the pointer — the core reversal

Make curr point to prev instead of forward. The first node points to null (prev starts as null). This is the magic step.

curr.next = prev // flip the arrow!
3

Advance prev forward

Move prev to the current node. It marks the front of the growing reversed chain.

prev = curr // advance prev
4

Advance curr using saved next

Jump curr to the saved node. Repeat from step 1 until curr is null — every node gets visited exactly once.

curr = next // move forward
5

🏁 Return prev — the new head

When curr is null, we're done. prev points to what was the last node — now the new head of the reversed list!

return prev // new head
Python
def reverse_linked_list(head):
    # Three pointers — that's all we need
    prev = None
    curr = head
 
    while curr is not None:
        next_node = curr.next    # 1. Save next
        curr.next = prev        # 2. Flip pointer
        prev = curr           # 3. Move prev
        curr = next_node      # 4. Move curr
 
    return prev  # new head of reversed list

04 — Complexity Analysis
Time & Space

This algorithm is a beautiful example of efficiency — one pass, zero extra memory.

O(n)

Time Complexity

We visit each of the n nodes exactly once. No nested loops, no backtracking — a clean linear scan.

O(1)

Space Complexity

Only 3 variables (prev, curr, next) no matter how large the list. We modify pointers in-place — no extra array or stack.

Real-World Applications
↩️

Undo/Redo

Text editors store history as a linked list. Reversal enables backward traversal through edits.

🌐

Browser History

Back/forward navigation uses linked structures — reversal is at the heart of going backward.

🧮

Big Numbers

Digits stored as a list — reversal aligns digits for arithmetic operations on huge numbers.

🧬

DNA Processing

Bioinformatics processes complement strands — often the reverse of the original DNA sequence.

🗂️

Stack Reversal

A stack is a linked list under the hood. Reversing it flips the LIFO order — used in schedulers.

🎮

Game Replays

Move histories stored as linked lists — reversal enables "watch your game in reverse" features.
